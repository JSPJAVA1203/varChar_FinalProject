관리자 페이지 기능

1) 상품 추가/(재고, 가격)변경/삭제

1-1. 상품 추가 : 
상품명 -> 가격 -> 재고 -> 카테고리 (select 형식) -> 사진 업로드 -> 설명 순으로 <input>태그안에 입력한후
submit. 상품명부터 설명까지 <form>으로 묶여있는 상태

1-2. 상품 삭제:
ajax를 이용한 이중 카테고리 (상위 카테고리를 선택하면 하위의 카테고리가 나오는 형식 -> 본 페이지에서는
select 형식의 상위 카테고리를 선택시 그 카테고리에 해당하는 상품이 select 형식으로 출력된다)

화면에 상품 PK, 상품명, 가격, 재고, 사진이 disabled readonly 형식으로 참고만 가능하도록 출력된다.
Submit을 누르면 해당 상품이 삭제됨. (버튼 클릭시 다시 한번 확인하는 confirm()추가 예정)

1-3. 상품 수정 (재고/가격)
상품 삭제와 마찬가지로 ajax를 이용한 이중 카테고리 (삭제와 내용 역시 동일)
화면에 상품 PK와 상품명 출력.
본 수정 폼에서는 기존 재고와 가격을 가독성이 더 좋게 하기 위해 하단에 따로 재고와 가격을 기재함.
ex) ※ $-{teaData.teaName} 상품의 기존 재고는 $-{teaData.teaCnt}개, 기존 가격은 $-{teaData.teaPrice}원 입니다.

변경할 목록에서 <input type="radio"> 형식으로 모두, 재고만, 가격만 총 3가지의 옵션중 하나 선택 가능
해당 페이지 로드시 디폴트로 모두에 체크됨
재고 선택시 가격 변경란 disabled로 입력 불가
가격 선택시 재고 변경란 disabled로 입력 불가
모두 선택시 두 변경란 모두 입력 가능.
역시 submit을 누르면 <form>에 담겨서 전송됨.

2) 상품 + 후기 해시태그 관리

상품 및 후기 해시태그 관리는 모두 로직은 동일

차이점 : 상품 해시태그 관리시 -> ajax를 사용한 이중 카테고리 (상품 관리의 수정/삭제와 동일)
           후기 해시태그 관리시 -> ... 이건 접근방법을 좀 더 논의 해봐야할듯

카테고리 클릭 -> 상품 클릭시 해당 상품의 해시태그가 <input type="button" value="해시태그"> 이런식으로 
최대 3개까지 출력됨
해시태그를 추가하고 싶다면 해시태그 직접 입력을 눌러서 유효성 검사를 거쳐 정상적인 값의 해시태그를 <form>
에 씌워진 공간에 담음
그리고 기존의 해시태그 버튼을 클릭시 수정/삭제 버튼 출력 -> 수정시 유효성을 거쳐서 value값을 바꿔주고, 삭제시
confirm()을 통한 여부를 한번 더 확인후 <form>안에서 삭제.
